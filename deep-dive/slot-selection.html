🔀 Slot Selection (A/B)

Reads slot metadata and selects the healthy, active slot — enabling seamless OTA updates with instant rollback.

📋 Overview

  • A/B partitions: Every critical partition exists in two copies: system_a/system_b, vendor_a/vendor_b, etc.
  • Seamless updates: OTA writes to the inactive slot while the device runs normally. On reboot, the bootloader switches to the updated slot.
  • Rollback safety: If the new slot fails to boot (after 7 attempts), it's marked unbootable and the bootloader falls back to the previous slot.

⚙️ How It Works

Read Boot Control Metadata

ABL reads the boot control structure from the misc partition. This contains per-slot metadata: priority, tries_remaining, successful_boot, active.

Evaluate Slot Health

For each slot, check: Is it marked bootable? Does it have tries_remaining > 0 or successful_boot == true? Unbootable slots are skipped.

Select Highest-Priority Bootable Slot

The slot with the highest priority that is still bootable is selected. After an OTA, the newly written slot gets the highest priority.

Decrement tries_remaining

If successful_boot is not yet set, decrement tries_remaining. When it reaches 0 and successful_boot is still false, mark the slot unbootable.

Set Kernel Slot Suffix

Pass androidboot.slot_suffix=_a (or _b) to the kernel command line. All partition references use this suffix.

🔗 A/B Slot Decision Flow

📁 Key Files & Artifacts

File / ArtifactPurpose
misc partitionStores boot control metadata (slot A/B state, retries, success flags)
android.hardware.boot HALAbstraction layer for reading/writing slot metadata from userspace
bootctlCommand-line tool to query/set slot status from Android shell
update_engineOTA daemon that writes to the inactive slot and marks it for boot

⚠️ Common Checks & Failure Modes

🚫 Both Slots Unbootable

Both slots exhausted their retry count without successful_boot. Device enters fastboot/recovery — requires manual reflash.

⚡ OTA Slot Not Marked Active

If update_engine crashes before marking the new slot active, the device continues booting from the old slot on next reboot.

💻 Code Snippets

bash 
# Check current slot
adb shell getprop ro.boot.slot_suffix    # _a or _b

# Query slot status via bootctl
adb shell bootctl get-current-slot
adb shell bootctl get-suffix 0           # _a
adb shell bootctl is-slot-bootable 0     # 1 = yes
adb shell bootctl is-slot-marked-successful 0

# Via fastboot
fastboot getvar current-slot
fastboot getvar slot-count
fastboot --set-active=b                  # Switch to slot B
