💾 Mount Filesystem Images

Dynamic partitions carved from the super partition are mounted as the root filesystem — enabling the full Android userspace.

📋 Overview

  • Dynamic partitions: Since Android 10, all system images live as logical volumes inside a single physical super partition, managed by dm-linear.
  • Mount sequence: First-stage init mounts early partitions (/system, /vendor); second-stage init mounts the rest and applies overlays.
  • Root switch: After mounting, the root filesystem switches from ramdisk to /system (system-as-root).

⚙️ How It Works

Parse Partition Metadata

Read the LP (Logical Partition) metadata from the first 4 KiB of the super partition. This describes the layout: extent ranges, partition names, group constraints.

Create dm-linear Targets

For each logical partition, fs_mgr creates a device-mapper linear target that maps a contiguous virtual block device to physical extents within super.

Layer dm-verity

On top of each dm-linear device, dm-verity is added (if the partition has an AVB hashtree descriptor). This makes reads go through: dm-verity → dm-linear → physical block.

Mount Partitions

Mount each verified device at its mount point: /system, /vendor, /product, /system_ext, /odm. All mounted read-only with barrier=1.

Apply Overlays & Switch Root

Vendor overlays (e.g., runtime resource overlays on /vendor) are applied. Root filesystem switches from ramdisk to /system (system-as-root since Android 10).

⚠️ Common Checks & Failure Modes

🚫 LP Metadata Corrupt

If logical partition metadata in super is corrupt, no partitions can be carved. Device fails to mount /system and boot stalls.

⚡ Filesystem Mismatch

fstab says ext4 but the partition is formatted as f2fs (or vice versa). Mount fails with "wrong fs type" error.

💻 Code Snippets

bash 
# View mounted partitions
adb shell mount | grep -E "system|vendor|product|odm"

# Check dm device mapping
adb shell dmctl table system
adb shell dmctl table vendor

# View logical partition layout
adb shell lpdump    # Shows super partition metadata

# Check dynamic partition groups
adb shell cat /dev/block/by-name/super | head -c 4096 | xxd | head
