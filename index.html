🔐 ABL — Android Bootloader
  • Power-on reset → SoC ROM loads ABL from boot partition.
  • Verifies boot chain integrity via vbmeta signature.
  • Reads A/B slot metadata; selects active slot; loads kernel + ramdisk.
Key files & artifacts vbmeta.img, avbtool, bootloader log in /proc/last_kmsg.
ABL is SoC-vendor-specific (Qualcomm XBL/ABL, MediaTek preloader, etc.). Slot metadata lives in misc partition.

⚙️ Linux Kernel
  • Decompresses kernel image (GKI or vendor kernel) into memory.
  • Initializes core subsystems: scheduler, memory, interrupts, drivers.
  • Mounts initial ramdisk (initramfs) as temporary root filesystem.
Details GKI (Generic Kernel Image) since Android 12. Kernel command-line passes androidboot.* params. Device-tree blob (DTB) provides hardware description.

📜 init — First Stage (Ramdisk)
  • Kernel executes /init (PID 1) from ramdisk — first userspace process.
  • Parses early init.rc and device-specific init.<hw>.rc files.
  • Reads fstab.qcom (or fstab.<hw>) to discover partition layout.
Vendor-specific note fstab.qcom is Qualcomm-specific. Other SoCs use fstab.<hardware>. First-stage init runs in ramdisk before real root is mounted. Triggers: early-init, init, late-init.

🛡️ AVB — Android Verified Boot 2.0
  • Validates vbmeta chain — ensures partitions haven't been tampered with.
  • Sets up dm-linear device-mapper targets to carve logical partitions from super.
  • Configures dm-verity for block-level integrity verification at read time.
Verification flow avbtool generates vbmeta with RSA-4096/SHA-256 signatures. Hash descriptors are chained across partitions. On failure → boot aborts or falls back to recovery, depending on device policy (eio/restart/none).

🌳 Merkle Hash Tree Verification
  • dm-verity reads hash_tree_descriptor from vbmeta.
  • Builds in-memory Merkle tree; validates each 4 KiB block on access.
  • Corruption detected → I/O error returned to caller (EIO mode).
How it works Hash tree root is signed in vbmeta. Each leaf = SHA-256 of a 4K data block. Tree is verified lazily (on read) — no full-image scan at boot.

🚫 Recovery / Abort
  • Tampered or corrupt partition detected — boot halted.
  • Device reboots into recovery mode or displays warning.
  • Error mode set by avb_handle→status: eio | restart | none.
Error handling On unlocked bootloader: warning shown, boot continues. On locked bootloader: boot aborts. Slot may be marked unbootable; bootloader tries other slot.
verified → slot selection

🔀 Slot Selection (A/B)
  • Reads slot metadata from misc partition (boot control HAL).
  • Checks successful_boot, unbootable, retry_count flags per slot.
  • Selects healthy slot (A or B); marks current slot as "active" for next boot.
A/B seamless updates Two copies of every partition (system_a/system_b, etc.). OTA updates write to inactive slot; on reboot, bootloader switches. If boot fails 7 times → slot marked unbootable, fallback to other slot.

💾 Mount Filesystem Images
  • Dynamic partitions carved from super via dm-linear → logical volumes.
  • Mounts: /system, /vendor, /odm, /product, /system_ext.
  • Overlay mounts applied (e.g., vendor overlay on /vendor); switches root to /system.
Partition details super partition holds all dynamic partitions as logical volumes (LVM-style). fstab declares mount options: ro, barrier=1, verify, etc. First-stage init handles early mounts; second-stage init handles the rest.
filesystem ready → SELinux

🔒 SELinux / sepolicy Setup
  • Loads compiled SELinux policy from /vendor/etc/selinux/ + /system/etc/selinux/.
  • Sets SELinux to enforcing mode; labels all mounted filesystems.
  • Re-executes /init in proper SELinux domain (u:r:init:s0) — "second-stage init".
Policy compilation CIL fragments from system + vendor merged at build time. selinux_setup runs between first and second stage init. File contexts applied via restorecon recursively.

🚀 Second-Stage init — Service Startup
  • Parses full init.rc tree; fires triggers: post-fs, boot, late-init.
  • Starts class core: ueventd, logd, servicemanager, hwservicemanager.
  • Starts class hal / main: HAL daemons, vold, surfaceflinger, netd.
Service classes (init.rc) class core — essential daemons that run before main.
class main — standard system services.
class late_start — non-critical services after boot complete.
on property:sys.boot_completed=1 triggers late_start.
class main → Zygote fork

🧬 Zygote — Android Runtime Origin
  • Started by init as service zygote (app_process64 / app_process32).
  • Preloads Android framework classes + resources; starts SystemServer.
  • Listens on UNIX socket; forks new app processes via fork() + specialize().
Zygote details ZygoteInit.main() → preloads ~8000 classes, shared memory pages.
SystemServer starts AMS, PMS, WMS, and 100+ system services.
Boot animation stops when sys.boot_completed=1.
First app launched: Launcher (home screen).
